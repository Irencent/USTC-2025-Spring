B站“低创“视频创作者识别¶

狗熊会¶

为了吸引更多的视频创作者，提供更多种类、质量更好的作品，B站于2018年提出了创作者激励计划。然而，为了进入激励计划，部分用户通过对网络素材的简单拼接和编辑来大规模生产视频，从而快速获得高播放量。这些用户被称为“低创”用户。“低创”用户的出现使得B站视频质量严重下降。因此，本案例将采用聚类方法对B站用户进行细分，在识别出“低创”用户的同时深入了解其他用户的特点，从而对用户做到“精准激励”，达到视频质量和内容升级的目的。

案例中所使用的数据是从B站官网爬取的用户数据，共35,157条，每一条数据代表一个用户信息，每条用户信息包含16个变量

本案例的学习目标包括：

（1）学习使用聚类分析的方法对用户进行细分

（2）学习如何在Python中对数据进行描述分析

（3）学习如何在Python中进行k-prototypes聚类分析

data1 = pd.read_csv('data.txt') # 读入数据


##  数据处理（因子化）

def data_process(data1):
    
    # 性别因子化
    encoder = preprocessing.LabelEncoder()  # 将encoder代表数值编码的函数（会将类别转为正整数）
    data1['sex'] = encoder.fit_transform(data1['sex']) # 进行转化
    data1['sex'] = data1['sex'].astype('object') # 将变量sex变为因子变量
    
    # 生日因子化
    f = lambda x:x[:2]  # 由于生日原始格式为09-19 这里我们只要月份信息，因此只取前两个即可
    data1['birthday'] = data1['birthday'].apply(f)  # 将函数f应用到birthday的每一条数据
    data1['birthday'] = data1['birthday'].astype(int).astype('object') # 数据原本格式为字符格式，这里先转为整型再转为因子
    
    return data1


data1 = data_process(data1)

任务四：建立聚类模型¶

选取聚类主题数K=4，建立聚类模型。在进行聚类时，我们使用的变量包括：“性别”、“生日”、“等级”、“种类“、”视频数量“、“相册数量”、“文章数量”、“总作品数量”、“粉丝人数”、“关注人数“、“获赞数量”、“总播放量”、“平均播放量”、“获赞率”、“粉丝留存率”16个变量，由于变量中既包含离散型变量，又包含连续型变量，因此使用k-prototypes聚类方法，你可以通过调用kmodes.kprototypes 里面的KPrototypes函数来建立该模型。

# 聚类之前，将存在空值的数据去掉（之前填充的只有获赞量与播放量）
data1=data1.dropna(axis=0,how='any')

# 三类变量需要不同的处理方式，以下三个变量用于存储他们的名称以便之后提取这些变量
# 离散变量名称（类别型）
qualitative = ['sex','types','level','birthday']
# 连续变量名称（计数型）
numerical = ['following','fans','view','likes','album','article','video','audio','sum']
# 连续变量名称
continuous = ['wpr','ftr','apr']

# 提取三类变量
data_qual = data1[qualitative]
data_nume = data1[numerical]
data_conti = data1[continuous]

# 将离散变量转为int型（聚类模型要求离散变量的格式）
ints = lambda x:x.astype(int) # ints函数将数据列x转为int型
data_qual = data_qual.apply(ints) # apply函数使ints函数遍历data_qual中的列
# 将整数变量与连续变量均采取对数变换
log = lambda x:np.log(x+1) # log函数 将数据列x的每一个数据转为ln(x_i+1) 因为data_nume为计数型的数据，存在0值，所以这里+1
logs = lambda x:np.log(x)  # logs函数 将数据列x的每一个数据转为ln(x_i)
data_nume = data_nume.apply(log)
data_conti = data_conti.apply(logs)
# 连接三类数据，并重置数据索引
data = pd.concat([data_conti,data_nume,data_qual],axis=1) # 这里将离散变量连在了最后，列的索引为12，13，14，15
data.index = range(len(data)) # 重置索引


# 定义模型 4个类别，算法采用Huang提出的算法，n_jobs代表模型运行时调用所有的计算机cpu能使用的线程（一般情况下速度会变快）
kp = KPrototypes(n_clusters=4, init='Huang',n_jobs=-1)
# 指定离散变量位置（位置用的索引），并训练模型
clusters = kp.fit_predict(data,categorical=[12,13,14,15])

【解读参考】：

第一类用户呈现高播放量，低粉丝数，低获赞率，低粉丝留存率的特点，符合之前分析的大部分“低创”用户的特点，其粉丝留存率与获赞率并不高，将这一类用户称为“低创“用户。

第二类用户是具有低播放量，高粉丝量，高获赞率，高粉丝留存率的特点，这说明该类用户创作的作品质量高，能够用较少的播放量即可吸引较多的粉丝，将这一类用户称为高质量用户。

第三类用户具有超高播放量，超高粉丝量。其作品数量较多，质量也呈现了参差不齐的情况，因而这一类用户在获赞率与粉丝留存率上并不一定很高，将这一类用户称为 up 主用户。

第四类用户介于三者之间，构成较为复杂，实际上是一种过渡的用户，不适于对这类用户进行简单地归类，这里将这一类用户称为过渡用户。
